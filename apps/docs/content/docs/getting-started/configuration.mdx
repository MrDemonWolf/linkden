# Configuration

LinkDen keeps its `.env` file minimal on purpose. You only need four environment variables to get running. Everything else -- email, CAPTCHA, branding, and more -- is configured through the admin panel and stored in the database.

## Environment Variables

Create your `.env` file from the example:

```bash
cp .env.example .env
```

Then set these values:

| Variable | What it does | Example |
|----------|-------------|---------|
| `BETTER_AUTH_SECRET` | Signs session cookies (keep this private!) | `openssl rand -base64 32` |
| `BETTER_AUTH_URL` | URL where the API server runs | `http://localhost:3000` |
| `CORS_ORIGIN` | URL where the web frontend runs | `http://localhost:3001` |
| `NEXT_PUBLIC_SERVER_URL` | Public API URL (usually same as `BETTER_AUTH_URL`) | `http://localhost:3000` |

### Generating a Secret

Run this to generate a strong, random secret:

```bash
openssl rand -base64 32
```

Paste the output as your `BETTER_AUTH_SECRET`. Use a different secret for production.

### Local Development

```bash
BETTER_AUTH_SECRET=your-generated-secret
BETTER_AUTH_URL=http://localhost:3000
CORS_ORIGIN=http://localhost:3001
NEXT_PUBLIC_SERVER_URL=http://localhost:3000
```

### Production

```bash
BETTER_AUTH_SECRET=your-production-secret
BETTER_AUTH_URL=https://api.yourdomain.com
CORS_ORIGIN=https://yourdomain.com
NEXT_PUBLIC_SERVER_URL=https://api.yourdomain.com
```

> **Tip:** `BETTER_AUTH_URL` and `NEXT_PUBLIC_SERVER_URL` should point to the same server. `CORS_ORIGIN` points to your frontend.

## Authentication

LinkDen uses email and password authentication. No external auth service or OAuth setup is needed.

During the [setup wizard](/docs/getting-started/quick-start), you create your admin account. After that, sign in at `/admin/login` with the same email and password. All admin pages are protected -- unauthenticated visitors only see your public link page.

## Admin Settings (Runtime Config)

Most configuration lives in the admin panel at `/admin/settings`, not in environment variables. This means you can change settings without redeploying:

- **Email provider** -- Resend or Cloudflare Email Workers.
- **CAPTCHA** -- Cloudflare Turnstile or Google reCAPTCHA.
- **Branding** -- Whitelabel footer text and link.
- **vCard** -- Enable or disable the downloadable contact card.
- **Apple Wallet** -- Team ID, Pass Type ID, and custom QR URL.
- **SEO** -- Page title, meta description, and OG image.
